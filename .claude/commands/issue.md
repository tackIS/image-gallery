# GitHub Issue 作成

ユーザーの説明をもとに GitHub Issue を作成してください。

## 手順

1. `gh label list` で既存のラベルを確認
2. ユーザーに以下を確認（不明な場合）:
   - Issue の種類: 機能追加 / バグ修正 / 改善 / リファクタリング
   - 概要と詳細

3. 以下のフォーマットで Issue を作成:

```
gh issue create \
  --title "<type>: <タイトル>" \
  --body "## 概要
<概要>

## 詳細
<詳細説明>

## 完了条件
- [ ] <条件1>
- [ ] <条件2>
" \
  --label "<適切なラベル>"
```

### type の選択基準
- `機能`: 新しい機能の追加
- `バグ`: 不具合の修正
- `改善`: 既存機能の改善
- `リファクタリング`: コード品質の向上

4. 作成した Issue の URL を報告
