# 変更セルフレビュー

コミット前の変更をレビューし、問題点を報告します。

## 手順

1. 変更差分を確認:
```bash
git diff
git diff --cached
git status
```

2. 以下の観点でレビュー:

### 必須チェック
- **型安全性**: `any` の使用、型アサーションの妥当性
- **ダークモード**: `dark:` プレフィックスの付け忘れ
- **エラーハンドリング**: try-catch、Result型の処理漏れ
- **Tailwind v4準拠**: v3構文の混入がないか

### 推奨チェック
- **パフォーマンス**: 不要な再レンダリング、メモ化の欠如
- **アクセシビリティ**: ボタンの aria-label、キーボード操作
- **セキュリティ**: コマンドインジェクション、XSS
- **命名**: 規約に沿った命名（PascalCase / camelCase / UPPER_SNAKE_CASE）

3. レビュー結果を報告:
   - 🔴 **必須修正**: 修正しないとバグやビルドエラーになる
   - 🟡 **推奨修正**: コード品質の改善
   - 🟢 **良い点**: 特に良い実装
